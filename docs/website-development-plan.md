# План разработки сайта Артёма Степанова

## 1. Executive Summary
Проект представляет собой разработку персонального сайта-визитки для Артёма Степанова, предпринимателя и эксперта по внедрению ИИ и автоматизации в бизнес-процессы. Сайт служит логическим продолжением офлайн-нетворкинга. Главная бизнес-задача проекта: конвертировать теплые офлайн-контакты в запросы на короткую диагностическую встречу, демонстрируя системный подход, глубокое понимание процессов и премиальное позиционирование.

## 2. Product Goal
**Главная цель:**
Мотивировать пользователя написать Артёму или назначить короткую встречу для обсуждения конкретной задачи бизнеса.

**Второстепенные цели:**
- Поддержать и развить контекст офлайн-знакомства (переход по QR-коду).
- Доступно объяснить пользу от автоматизации и ИИ (языком процессов, а не технологий).
- Правильно спозиционировать Артёма (бизнес-партнер, а не "программист по ТЗ").
- Отсеять нерелевантные запросы (желание "поиграть с ИИ" или отсутствие регулярных процессов).

## 3. Target Audience
**Аудитория:**
Владельцы бизнеса, руководители, предприниматели, обеспеченные люди.

**Контекст входа:**
Сайт открывается после личного знакомства офлайн (у бизнес-центра, в ресторане, фитнес-клубе и т.д.). Пользователь сканирует QR-код с визитки. У него уже есть первичное доверие к Артёму. Сайт должен быстро подкрепить это доверие профессиональным цифровым образом, "отзеркалить" бизнес-боли пользователя и предложить понятный следующий шаг.

## 4. Positioning
**Как мы позиционируем Артёма:**
- Предприниматель, который говорит на языке процессов, времени, клиентов, заявок и управляемости.
- Системный инженер с 15-летним опытом, для которого важны логика, точность и порядок.
- Специалист, который снимает рутину с владельца и команды.

**Кем Артём НЕ является:**
- Фрилансером.
- Разработчиком "чат-ботов".
- Техническим исполнителем (не пишет код "просто по ТЗ").
- Инфобизнесменом.

## 6. User Journey
1. **Знакомство офлайн:** Личный контакт.
2. **Переход:** Получение визитки → скан QR-кода.
3. **Открытие (Hero):** Моментальная загрузка, считывание заголовка. Понимание: "Он наводит порядок".
4. **Изучение:** Скролл → узнавание себя в блоке болей → просмотр решений → изучение обезличенных кейсов.
5. **Снятие барьера:** Понимание, что ТЗ не нужно, достаточно просто поговорить.
6. **Действие:** Выбор удобного мессенджера (Telegram) → отправка первого сообщения.
7. **Первая встреча:** Короткая диагностика процесса.

## 7. Content Requirements
Для начала сборки необходимо подготовить тексты:
- Заголовки и подзаголовки всех секций.
- 5-7 пунктов для списка "Где бизнес теряет время".
- Контент карточек (названия зон автоматизации + 1-2 предложения описания).
- Списки квалификации ("Подходит, если...", "Не подходит, если...").
- Тексты обезличенных кейсов в формате (Проблема - Решение - Результат).
- Текст "Об Артёме" (2-3 абзаца, фокус на системном мышлении).
- 5-7 вопросов и ответов для FAQ.
- Финальные контакты.
*(Напоминание: исключить "инфобизнес-язык", "ИИ-боты", "RAG-базы" и агрессивные обещания ROI).*

## 8. UI / UX Requirements
- **Мобильный контекст:** Сайт проектируется под QR-трафик после офлайн-знакомства. Пользователь может быть на улице, в машине, у бизнес-центра или в заведении, поэтому первые 3 блока должны быстро объяснять: кто Артём, чем полезен, почему стоит написать.
- **Первый экран:** Hero должен считываться в первые 5–7 секунд и комфортно помещаться на мобильном экране без ощущения пустоты. Не фиксировать высоту строго в 100vh.
- **Навигация:** Плавающая шапка (sticky) в минималистичном стиле (переход к форматам/контактам).
- **Карточки:** Без объемных теней. Легкие рамки (borders), темная заливка, строгая геометрия.
- **Кейсы:** На мобильном — вертикальный список карточек. На десктопе — сетка 2–3 колонки. Не прятать кейсы в карусель, чтобы не снижать доверие.
- **CTA-кнопки:** Удобные для мобильных устройств (высота от 48px). Заметные, но не кричащие.
- **Мобильная версия (Mobile-First):** Основной приоритет, так как основной трафик будет с визиток. Текст должен быть крупным и читаемым.
- **Анимации:** Мягкие `fade-in` при скролле. Микро-взаимодействия на кнопках и карточках. Без сложных визуальных наворотов.
- **Скорость:** Мгновенное открытие после сканирования QR.

## 9. Visual Direction
- **Общий тон:** Тёмный премиальный AI, строгий, дорогой, технологичный, легкая бизнес-дерзость (уверенность).
- **Цветовая палитра:** 
  - Фон: Глубокий черный (`#000000` или `#0A0A0A`).
  - Текст: Оттенки серого и белого (сохраняя контраст).
  - Акценты: Сдержанный монохром или один строгий цвет без неонового свечения.
- **Типографика:** Современные геометрические гротески (Inter, SF Pro, Roboto, Manrope).
- **Сетка и воздух:** Большие отступы. Минимум визуального шума. Никаких лишних элементов.
- **Запрещено:** Дешевые стоки, киберпанк-неон, стилистика дешевых лендингов, перегруженность.

## 10. Suggested Tech Stack
**Рекомендация: Astro + Tailwind CSS.**
- **Почему:** Одностраничный статический сайт-визитка, QR-трафик, нужна максимальная скорость, минимум JS и простая статическая отдача. Astro идеально решает эту задачу, предоставляя компонентный подход во время разработки и нулевой JS в бандле для пользователей.
- *(Альтернатива: Next.js (App Router, SSG) — рекомендуется оставить на будущее, если появится блог, CMS, сложная динамика или серверные функции).*

## 13. Data / Content Model
Весь контент необходимо вынести в конфигурационные файлы (например, `/src/config/content.ts`), чтобы изолировать данные от логики компонентов:
- `HERO_CONTENT`: заголовки, подзаголовки.
- `PAIN_POINTS`: массив строк.
- `AUTOMATION_AREAS`: `[{ id, title, description }]`
- `CASES`: `[{ id, title, problem, solution, result }]`
- `FAQ_ITEMS`: `[{ question, answer }]`
- `CONTACTS`: ссылки на TG, MAX, Phone.

## 15. SEO Requirements
Базовая гигиена (глубокое SEO не нужно для QR-трафика):
- **Title:** Артём Степанов | ИИ и автоматизация бизнес-процессов
- **Description:** Помогаю бизнесу навести порядок в заявках, поддержке и рутине. Инженерный подход к автоматизации.
- **H1:** "ИИ и автоматизация для процессов..." (только 1 на страницу).
- **Open Graph:** Строгое стильное превью (og:image), чтобы при отправке в мессенджеры ссылка выглядела премиально.

## 16. Analytics Requirements
События для отслеживания:
- Клики по кнопкам связи (Telegram, Phone, MAX).
- Глубина скролла (доходят ли до кейсов/FAQ).
- (Опционально) UTM-метки для QR-кода визитки `?utm_source=business_card`.

## 17. Performance Requirements
- **LCP (Largest Contentful Paint):** < 1.5 сек.
- Отложенная загрузка тяжелых элементов ниже первого экрана.
- Оптимизация медиафайлов (WebP/AVIF).
- Минимум JS в бандле (Астро обеспечивает это из коробки).

## 18. Deployment Plan
**Рекомендация: Cloudflare Pages или Vercel.**
- Идеальная совместимость со статическими сайтами (Astro).
- Быстрый CDN для моментальной отдачи статики в любой точке.
- Простая настройка CI/CD из GitHub.
- Бесплатен для MVP.

## 19. Development Stages
1. **Контент и структура.** Утверждение всех текстов.
2. **Дизайн-концепция.** Настройка цветов, шрифтов, UI-кит (Tailwind).
3. **Верстка MVP.** Сборка всех секций без контента (скелет).
4. **Интеграция контента и адаптив.** Подключение данных, полировка мобильной версии.
5. **Настройка связей.** OG-теги, контакты, базовая аналитика.
6. **Тестирование.** Проверка скорости и отображения.
7. **Деплой.** Привязка домена и релиз.

## 20. Acceptance Criteria (Критерии приемки)
- Сайт открывается мгновенно после сканирования QR-кода.
- Безупречное отображение на мобильных устройствах (без горизонтального скролла).
- Основной смысл считывается за первые 5-7 секунд.
- Нет запрещенной терминологии ("RAG-базы", агрессивных продаж).
- Контакты корректно открывают системные приложения.
- Присутствует премиальное "дорогое" ощущение интерфейса.

## 21. Risks and Open Questions
**Требует решения до начала написания кода:**
1. **Контент:** Предоставить точные факты для кейсов (без нарушения NDA) и текст для блока "Об Артёме".
2. **Медиа:** Желательно иметь качественную фотографию Артёма, но MVP можно запустить и без фото, если визуальная система, типографика и тексты достаточно сильные.
3. **Домен:** Выбрать доменное имя.
4. **Контакты:** Уточнить финальный телефон, Telegram-ссылку и MAX-ссылку.
5. **Форма заявки:** Для MVP форму не использовать, но оставить как возможное расширение.
6. **Блог:** Для MVP не нужен. Возможен позже, если появится стратегия контента.
7. **Аналитика:** Решить, использовать ли Яндекс.Метрику, Google Analytics или лёгкую privacy-friendly аналитику.
8. **Кейсы:** Уточнить, какие детали можно раскрывать публично без нарушения конфиденциальности.
